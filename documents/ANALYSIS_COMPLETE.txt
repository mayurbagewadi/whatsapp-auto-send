ARCHITECTURE OVERVIEW
================================================================================

MEDIA UPLOAD PIPELINE (9 PHASES):

1. File Selection   -> User selects via file picker or Google Drive
2. Validation       -> MIME type, size, quota checks
3. Encoding         -> Convert to base64 data URI
4. Encryption       -> AES-256-GCM with random IV
5. Redux Update     -> Add to media[] state
6. Local Storage    -> Persist to IndexedDB
7. Attachment UI    -> Click button, select menu, fill input
8. WhatsApp Send    -> Click send, poll for confirmation
9. Backend Logging  -> POST status to /update-msg-report endpoint

STORAGE LAYERS:

Local:
  - Chrome storage.local (userData, report_status)
  - IndexedDB SheetWA_DB (retryRequests, reportStatus, mediaCache)

Remote:
  - Backend API (https://backend.sheetwa.com)
  - Supabase PostgreSQL (auth, media_uploads, quotas, logs)

ENCRYPTION:
  Algorithm: AES-256-GCM
  IV: 128-bit random per file
  Key: PBKDF2 derived from JWT token
  Scope: Session-based (lost on logout)
